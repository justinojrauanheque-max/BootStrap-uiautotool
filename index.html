Getting started

Workflow guide

Auto Clicker - Form Helper keeps three independent automation surfaces in one workspace: Click and Fill for table-driven actions, UserScript for custom JavaScript, and OCR for visual targets that are easier to recognize on screen.

Click and Fill UserScript OCR
1. Choose a sessionPick the automation surface that fits the job.
2. Save the targetSet a URL, script metadata, or OCR capture.
3. Enable executionRun only after the rule is tested on the target page.

Auto Clicker - Form Helper automates clicks and form filling on websites and iframes. Create one configuration per site, define actions, and let the extension run the flow automatically.

  • Create a configuration for the target URL.
  • Add actions (click or fill) with interval and repeat.
  • Enable the configuration and follow along with FloatBox.

Installation

Install from the Chrome Web Store: Open in Chrome Web Store. Then pin the extension icon for quick access.

After installing, open and set the target URL.

Create a configuration

Each configuration represents a site or flow. Set a name, the URL, and an initial delay if the page needs time to load. The manual Add action button requires the configuration name and URL first, while right-click capture can still create rows from the target page.

NameUse names like Login, Checkout, Lead form, or Daily report.
URLUse the page where the routine starts. Wildcards are supported by the matching logic.
Initial delayAdd a delay when the page needs time to load before the first action.
ActionsEach row has its own selector, mode, interval, repeat, fill value, and advanced action settings.
  • Name: identify the flow (e.g., Signup, Login, Checkout).
  • URL: the main page where automation starts.
  • Initial delay: time before the first action.

After saving, add actions in .

Capture elements

Use the most stable target available. Prefer IDs, data attributes, labels, and compact CSS selectors when the site provides them; use OCR when the visual target is stable but the DOM selector changes.

Right click captureLoad XPath or CSS from the extension context menu.
Manual selectorPaste a known CSS selector or XPath directly in the action row.
OCR captureDrag over a visual button, text, checkbox, or switch.

You can capture selectors in two ways:

  • Automatic: right-click the element and use “Load Element”.
  • Manual: paste the XPath or CSS selector directly in the field.

If the element is dynamic, see .

Action types

ClickClicks buttons, links, tabs, and controls.
FillWrites text into inputs, textareas, and compatible editable controls.
Double clickAvailable in OCR rules for targets that require a double activation.
Check / switchAvailable in OCR rules for checkbox and switch-like visual targets.

The extension supports click and fill actions in Click and Fill, plus visual actions in OCR. Each action owns its own configuration so changing one row does not change the others.

  • Click: clicks the matched element.
  • Fill: pastes or types values into inputs.
  • Paste/Type: choose the method in the edit modal.
  • Default/Mutation Observe: Default runs by selector; Mutation Observe waits for dynamic DOM changes when Advanced + Force DOM are enabled.

For timing and repetition, see .

Interval & repeat

Control how many times each action runs and the time between runs.

  • Interval (ms): time between repeats.
  • Repeat: fixed number, -2 for infinite loop, or 1 to apply to all matched elements.

If you need more control, go to .

Iframes & dynamic pages

The extension works on dynamic pages and iframes. When the element isn't in the DOM immediately, use:

  • Mutation Observe: detects when the element appears.
  • Sandbox “Force DOM”: improves capture on restricted pages.

These options are available in .

Advanced mode

Enable advanced mode to unlock extra options, including Mutation Observe and sandbox controls. These controls apply to Click and Fill and to the UserScript editor settings panel, while OCR keeps its capture settings inside the OCR section.

  • Mutation Observe: actions that wait for the element to appear.
  • Sandbox: choose Default or Force DOM.

UserScripts

Use the UserScript editor for advanced automations and routines that are easier to express in JavaScript. UserScript is independent from Click and Fill and OCR: when this session is active, the active script can run; when another session is active, script registration is turned off.

Script listCreate, select, import, export, and delete scripts from the left session list.
Default templateThe untouched template is only a starting point. Edit it before saving or injecting.
Editor menuFile, Edit, Search, Run, and Tools provide save, format, find, validate, import, export, and run commands.
Apply timingChoose whether scripts apply after page reload or immediately on already-open tabs.

The right settings panel for UserScript contains three groups: Editor options for theme, size, keymap, indentation, wrapping, syntax check, and save-on-blur; Execution options for active-script-only behavior, sandbox mode, and apply timing; Security options for the blacklist.

OCR automation

OCR mode is independent from Click and Fill and UserScript. It is designed for visual targets: buttons, labels, checkboxes, switches, scroll areas, and text that can be identified from a small capture area.

ConfigurationSave the OCR configuration name, target URL, and initial delay before capturing.
CaptureOn the configured URL, use the FloatBox plus button or Ctrl+Shift+Y to capture a visual target.
MatchThe rule stores visible text, a selector hint, and the capture rectangle for fallback matching.
ActionsClick, double click, scroll, fill, and check / switch are available.
VigilanteOCR rules can wait for dynamic targets before acting.
Apply timingBy default the FloatBox appears after the target page reloads. Live injection can be enabled when needed.

The OCR list starts with only the header: Action, Vigilante, Interval, and Repeat. After a capture, each captured action appears with its own controls. Fill shows a fill value field, Scroll shows direction and amount, and Check / switch shows the target type.

The FloatBox controls are scoped to the target page. Plus arms a new capture, Play starts the configured OCR actions, Next rotates through captured actions, Preview highlights a target, Stop halts running timers, and Delete removes the selected captured action.

Keep captures tight. Smaller regions are easier to match and less likely to hit the wrong element after a layout change.

Blacklist

Define blocked sites to prevent accidental automation. Each blocked domain disables execution.

  • Add one domain per line.
  • Use it for sensitive areas (banks, social networks, etc.).

FloatBox feedback

FloatBox shows the action status in the corner of the page, including counters, element waiting, and completion.

  • Enable it in Settings → Feedback.
  • Great for debugging and validating the flow.

Save your configurations as JSON for backup or migration.

  • Export: generates a file with all configs.
  • Import: loads existing configurations.

Troubleshooting

Use this quick checklist:

  • Does the configuration URL match the current site?
  • Is the XPath/CSS selector still valid?
  • Does the element appear after load? Use Mutation Observe.
  • Is sandbox correct? Try Force DOM.
  • Is the domain blocked by blacklist?

Local-first policy

Privacy and permissions

The extension stores configurations locally in the browser and uses permissions only to run the automation features selected by the user.

Privacy policy

  1. Data collection and use: This extension is designed with your privacy in mind. It does not collect, store, monitor, transmit, or share any personally identifiable data or browsing information with the developer or third parties. All user-provided settings (such as XPaths, fill values, intervals, and repeat options) are stored exclusively in your browser’s local storage (via chrome.storage.local). This information is never sent to external servers or accessed by any other entity.
  2. Extension permissions: The permissions requested by the extension (storage, scripting, activeTab, tabs, webNavigation, contextMenus) are strictly necessary for its core purpose: automating clicks and form filling on web pages. None of these permissions are used to collect data unrelated to the extension’s functionality.
  3. Data security: Since all user data is stored locally on your device, security depends on the protections of your browser and operating system. The extension does not implement network security mechanisms because no data is transmitted.
  4. Changes to this privacy policy: We may update this privacy policy periodically. Any changes will be reflected on this page.
